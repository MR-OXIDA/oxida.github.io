批量修改域名解析记录

2020-05-12

操作场景

当您需要批量修改当前登录账户的域名解析记录时，本文档指导您如何批量修改域名解析记录。

操作步骤

1.登录DNSPod管理控制台。
2.在"DNS解析"列表中，选择【批量管理】，单击操作栏的【修改记录】，进入该域名的【修改记录】页面。如下图所示：

通过提供的搜索选项，查找您需要批量修改的解析类型，并单击搜索。

提供的搜索选项：

  • 记录类型：可通过记录类型搜索，如：A、CNAME、MX、TXT、NS、AAAA、SRV、URL。
  • 线路类型：可通过线路类型搜索，如：默认、电信、联通、教育网、搜索引擎、移动、境内、境外、奇虎、百度、谷歌、有道、必应、搜狗。
  • 主机记录：可通过填写主机记录相关关键字进行搜索。
  • 记录值：可通过填写记录值进行搜索。

例：选择记录类型与A记录，如下图所示：

注意：
最近添加的记录可能未包含在搜索范围中，如未搜索到请稍后重试。

3.在【修改记录】选项卡中,选择您需要修改的域名，并选择该域名下需要修改的主机记录。如下图所示：

4.选择您需要修改的记录，并选择或者填写需要修改内容，例：将"记录类型"修改为"CNAME",记录值:dnspod.cn。


提供的修改选项：

  • 记录类型：可修改为A、CNAME、MX、TXT、NS、AAAA、SRV、显性URL、隐性URL。

    • A记录：又称IP指向，用来指定主机名（或域名）对应的IP 地址记录。如果您需要实现一个或多个子域名批量指向同一个目标主机地址，您可以在“记录值”处填写目标主机地址。如：需要把选中的子域名“www.123.com” 和“cloud.123.com”的子域名IP指向“113.96.13.15”时，您可以在 “记录值” 处填写 “113.96.13.15” 即可。
    • CNAME：又称真实名称记录，用于将一个域名映射到另一个域名。如果您需要将一个或者多个子域名指向另一个域名，您可以在“记录值”处填写另一个域名。如：可以把“www.123.com”和“www.321.com”都指向域名“www.213.com”。
    • MX：又称邮件交换记录,用于电子邮件系统发邮件时根据收信人的地址后缀来定位邮件服务器，如果您需要将一个或者多个子域名指向同一个邮件服务器。您可以在“记录值”处填写邮件服务器的IP地址或企业邮局给您提供的域名。如：可以把“www.123.com”和“www.321.com”都指向域名邮件服务器“mxbiz1.qq.com”。同时可以批量设置MX优先级，用来指定邮件服务器接收邮件的先后顺序，数值越小优先级越高。
    • TXT：一般用于为某个主机名或域名设置说明。如果您需要对一个或着多个域名进行批量标识和说明，您可以在“记录值”处填写需要设置的标识和说明。
    • NS：又称域名服务器记录，用来指定该域名由哪个DNS服务器来进行解析。如果您需要将一个或者多个域名交给其他 DNS 服务商解析，您可以在“记录值”处填写需要托管的DNS 服务器域名。建议您使用腾讯云系统默认的 NS 记录，f1g1ns1.dnspod.net。
    • AAAA：用来将域名解析到IPv6地址的DNS记录。如果您需要批量将一个或者多个子域名指向同一个 IPv6 地址。您可以在“记录值”处填写需要解析的IPv6地址。
    • SRV：SRV 记录用来标识某台服务器使用了某个服务，常见于微软系统的目录管理。如果您需要批量修改一个或者多个子域名的SRV 记录。您可以在“记录值”处填写需要设置的SRV 记录。SRV 的记录值格式应为：优先级、空格、权重、空格、端口、空格、主机名，如：5 0 5269 example.domain.com。
    • 显性URL：用于将一个域名指向另外一个已经存在的站点时，需要添加 URL 记录。显性URL用的是301重定向技术，效果为浏览器地址栏输入“http://a.com”回车，打开网站内容是目标地址 “http://cloud.tencent.com”的网站内容，且地址栏显示目标地址“http://cloud.tencent.com”。如果您需要批量修改一个或者多个子域名指向同一个目标地址。您可以在“记录值”处填写需要设置的目标地址。
    • 隐性URL：隐性URL与显性URL的作用与操作方式相同。隐性URL用的是 iframe 框架技术、非重定向技术，效果为浏览器地址栏输入“http://a.com”回车，打开网站内容是目标地址“http://cloud.tencent.com”的网站内容，但地址栏显示当前地址 “http://a.com”。

    注意：
    目标地址不允许被嵌套时，则不能使用隐性转发（例如 QQ 空间，不能使用隐性转发）。

  • 线路类型：一般用于自动识别浏览者的来源，并把相同的域名智能DNS解析到双线路机器的网通或电信的IP， 以便就近访问您的网站。具体线路类型说明可参考：线路类型如何选择。如果您需要批量修改一个或者多个子域名的线路类型，选择您需要修改的线路类型即可。可修改的选项有默认、电信、联通、教育网、搜索引擎、移动、境内、境外、奇虎、百度、谷歌、有道、必应、搜狗。

  • 主机记录：用于修改子域名的前缀。如果您需要批量修改一个或者多个子域名的域名前缀，您可以在“记录值”处填写需要设置的域名前缀。

  • 记录值：用于修改解析记录中的记录值。如果您需要批量修改一个或者多个子域名的解析记录的记录值，您可以填写您需要修改的记录值。

  • TTL：指各地 DNS 服务器缓存解析记录的时长。增大TTL值，可以节约域名解析时间，给网站访问加速。减小TTL值，可以减少更换空间时的不可访问时间。同时修改TTL值可以提高系统的安全性。

4.单击【批量修改】，进入修改记录确认页面，检查并确认需要批量修改的内容无误后，单击【批量修改】，如下图所示：

5.，等待"状态"提示"成功"，至此批量修改域名解析记录完成。如下图所示。
