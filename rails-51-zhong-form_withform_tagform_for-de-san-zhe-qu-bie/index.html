Rails 5.1中form_with,form_tag,form_for的三者区别

ruby
2018-01-07

Rails 5.1中，form_tag和form_for已弃用，以后将由form_with替换。

这篇文章中，我将解释form_tag，form_for与form_with之间的差异，并用代码示例。

以前，当你要创建表单但没有基础模型时，可以使用form_tag。

<％= form_tag users_path do％> 
  <％= text_field_tag：email％> 
  <％= Submit_tag％> 
<％end％>

当你拥有模型时，使用form_for。

<%= form_for @user do |form| %>
  <%= form.text_field :email %>
  <%= form.submit %>
<% end %>
<％end％>

如你所见，我将form_for与builder field helpers一起使用，并没有使用form_tag。因此，这两种形式的语法都不同。
form_with并非是这样的，因为我们一直都在使用表单构建器。
form_with 没有模型：

<%= form_with url: users_path do |form| %>
  <%= form.text_field :email %>
  <%= form.submit %>
<% end %>

form_with与模型:

<%= form_with model: @user do |form| %>
  <%= form.text_field :email %>
  <%= form.submit %>
<% end %>

但你设置模型的参数时，scope和URL会自动从中派生。此行为类似于form_for。

自动ID和类消失了

form_tag和form_for为表单字段生成自动ID。 form_for也会对表单进行处理。

<%= form_for User.new do |form| %>
  <%= form.text_field :email %>
<% end %>

产生：

<form class="new_user" id="new_user" action="/users" ...>
  ...
  <input type="text" name="user[email]" id="user_email" />
</form>

使用form_with，你必须手动指定所有ID和类：

<%= form_with model: @user do |form| %>
  <%= form.text_field :name %>
  <%= form.text_field :email, id: :email, class: :email %>
<% end %>

产生：

<form action="/users" ...>
  ...
  <input type="text" name="user[name]" />
  <input id="email" class="email" type="text" name="user[email]" />  </form>

如果要使标签起作用，不要忘记为表单字段指定ID。

<%= form_with model: @user do |form| %>
  <%= form.label :name %>
  <%= form.text_field :name, id: :user_name %>
<% end %>

更新：从Rails 5.2开始，form_with将自动生成id，因此不再需要手动指定。

表单ID和类属性不再包装
之前：

<%= form_for @user, html: { id: :custom_id, class: :custom_class } do |form| %>
<% end %>

之后：

<%= form_for @user, html: { id: :custom_id, class: :custom_class } do |form| %>
<% end %>

表单字段不必与模型属性相对应

之前：

<%= form_for @user do |form| %>
  <%= form.text_field :email %>
  <%= check_box_tag :send_welcome_email %>
<% end %>

之后：

<%= form_with model: @user, local: true do |form| %>
  <%= form.text_field :email %>
  <%= form.check_box :send_welcome_email %>
  <%= form.submit %>
<% end %>

请注意，在后一个示例中，send_welcome_email的作用域将为User控制器参数的中：

params[:user][:send_welcome_email]

所以，如果需要，使用check_box_tag而不是form.check_box。

默认情况下，所有表单都是远程的

这个变化对我来说是最令我兴奋的。默认情况下，由form_with生成的所有表单都将由XHR（Ajax）请求提交。无需指定remote：true，就像使用form_tag和form_for一样。

我喜欢这个功能，因为它发送消息。多亏了Turbolinks，（默认）几乎所有GET请求都是XHR请求。多亏了form_with，这也可以成为表单的现实。只需要做一点工作（创建一个JavaScript响应模板），就可以消除所有整页刷新。

但是，如果要禁用远程表单，则可以使用local：true来完成。

<%= form_with model: @user, local: true %>
<% end %>

我没有在这篇文章中介绍所有内容，但是它足以让你开始使用form_with，因此不必再使用form_tag和form_for（无论怎么样它们都会被删除）。另外，可以阅读form_with的文档。
