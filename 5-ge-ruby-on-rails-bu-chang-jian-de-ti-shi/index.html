5个Ruby on Rails不常见的提示

Ruby on Rails
2019-03-15

自从我开始使用Ruby on Rails以来，经过几年的使用，我不禁想知道我为什么能在Ruby的世界中发现一些新东西。

Ruby很少使用必备的方法，主要是“语法糖”，虽然它很不规范，但它会使你的代码更清晰。不过有些时候可能只是新的Ruby或Rails版本出现了新的变化。

Hash#dig
我使用的第一个Ruby版本是1.8，但是这个方法是在2.3中引入的。
我做了多少次这样的事情：

if params[:user] && params[:user][:address] && params[:user][:address][:somewhere_deep]

后来才发现，对于Hash对象的&，可以将其dig视为一种安全的导航操作符。所以现在你可以重写这样的东西：

if params.dig(:user, :address, :somewhere_deep)

Object#presence_in

但你实际上不需要检查包含布尔运算的结果时，它允许你使用单个方法来调用替换条件（通常是三元运算符），但是检查对象本身。例如，你的代码如下所示：

sort_options = [:by_date, :by_title, :by_author]
...
sort = sort_options.include?(params[:sort]) 
  ? params[:sort] 
  : :by_date

# Another option
sort = (sort_options.include?(params[:sort]) && params[:sort]) || :by_date

这看起来不是更好吗？

params [：sort] .presence_in（sort_options）|| ：by_date

Module#alias_attribute

好吧，当我使用遗留数据库处理项目时，我发现这非常有用。它有一个表奇怪的列名一样SERNUM_0，ITMDES1_0等。我们将此表映射到ActiveRecord模型，而不是在其上编写查询和范围WeirdTable.where(SERNUM_0: ‘123’)，我们开始使用alias_attribute它，因为它不仅生成getter和setter（以及谓词方法），而且还在查询中工作：

alias_attribute：name，：ITMDES1_0 
... 
scope：by_name， - >（name）{where（name：name）}

Object#presence

这个比其他人更受欢迎。对ApiDock有很好的解释。所以，object.presence相当于：

object.present? ? object : nil

Module#delegate
由于某种原因，大多数开发人员仍然很少使用它。该方法的主要目的是松耦合并遵循得墨忒耳定律。想到这个主题的好文章之一是Avdi Grimm的“Demeter：这不仅仅是一个好主意。这是法律。“另外，在应用Demeter法则的背景下，查看关于利用的简短Rails最佳实践文章。下面的代码段也描述了这一点：delegate

class Profile < ApplicationRecord
  belongs_to :user
  delegate :email, to: :user
end
...
profile.email # equivalent to profile.user.email
